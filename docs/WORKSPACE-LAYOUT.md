# MetroMark Workspace Layout

Use this as a quick map for what to edit, what is generated, and what is local-only.

## Source Code (edit these)

- `public/`: frontend HTML, CSS, and browser JS.
- `server/`: Express API, Supabase data access, config.
- `scripts/`: operational scripts (harvest, backup, Windows helpers).
- `docs/`: runbooks and notes.
- `supabase/`: SQL migrations and schema helpers.
- `package.json`: project scripts and dependencies.
- `README.md`, `CHANGELOG.md`: project docs.

## Generated or Installed (do not hand-edit)

- `node_modules/`: npm-installed dependencies.
- `package-lock.json`: npm lockfile (regenerated by npm).

## Local Data and Backups (machine-specific)

- `data/`: local data output (ex: `data/backups/`, `data/metromark.db` if present).
- `.env.development`, `.env.production`: local secrets and environment config.

## Production Automation Helpers

- `scripts/windows/*.ps1`: wrappers for starting services, opening admin/app, and registering Task Scheduler jobs.

## Notes

- The repo path can live under any Windows user profile (Praji or Zing). Scripts resolve the repo folder relative to their own location.
- Local data and `.env` files are not meant to be shared publicly.
