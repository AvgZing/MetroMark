# MetroMark Roadmap Notes

This file captures near-term and long-term feature direction based on product intent.

## Near-Term (After MVP Stabilization)

- Supabase Auth + Postgres persistence migration.
- Better station grouping and dedup in dense areas.
- Manual override UI for name/location corrections.
- Route detail panel with per-line progress bars.
- User notes on stations.

## Mid-Term

- Frequency-based route filtering (day/night/weekend).
- Completion modes: passed through, boarded/deboarded, explored station, explored area.
- Goal lists and planning overlays.
- Issue reporting and feedback channel.

## Long-Term

- Fog-style tracking view with hex aggregation (H3).
- Optional passive movement logging with offline cache support.
- Custom route planning layers.
- Place pins/global notes layer.

## Non-Goals

- Global competitive leaderboard.
- Anti-cheat-heavy social platform mechanics.

MetroMark remains a personal-progress mapping tool first.
